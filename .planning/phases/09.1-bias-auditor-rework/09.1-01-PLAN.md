  <action>
Refactor the existing `run_audit()` function's inference loop to capture three additional data dimensions in a SINGLE pass (no second loop). All changes are inside `audit.py`.

**1. Per-source confusion tracking:**
Add a `source_stats: dict[str, dict]` alongside the existing `category_stats`. For each example, track `tp/fp/fn/tn/total` keyed by `ex.source_dataset`. After the loop, compute per-source metrics using existing `_compute_metrics()` plus false positive rate (`fp / (fp + tn)` with zero-division guard).

Build `per_source` list:
```python
per_source = []
for src, stats in source_stats.items():
    metrics = _compute_metrics(stats["tp"], stats["fp"], stats["fn"])
    fpr = stats["fp"] / (stats["fp"] + stats["tn"]) if (stats["fp"] + stats["tn"]) > 0 else 0.0
    per_source.append({
        "source": src,
        "total": stats["total"],
        **metrics,
        "false_positive_rate": round(fpr, 4),
    })
```

**2. Confidence score collection per category:**
Add a `category_confidences: dict[str, list[float]]` dict. For every example that has a `hate_type` (i.e., `gt_binary == "hate" and ex.hate_type is not None`), append `result["confidence"]` to the list keyed by `cat`. Also track a separate list `overall_confidences: list[float]` for all examples.

Build `confidence_dist` dict:
```python
confidence_dist = {}
for cat_value in HateType:
    cat = cat_value.value
    scores = category_confidences.get(cat, [])
    confidence_dist[cat] = {
        "scores": [round(s, 4) for s in scores],
        "count": len(scores),
    }
confidence_dist["_overall"] = {
    "scores": [round(s, 4) for s in overall_confidences],
    "count": len(overall_confidences),
}
```

**3. False positive sample capture:**
Add a `false_positive_samples: list[dict]` with `MAX_FP_SAMPLES = 10`. During the loop, when `gt_binary == "not_hate" and predicted == "hate"`, if `len(false_positive_samples) < MAX_FP_SAMPLES`, append:
```python

    "source_dataset": ex.source_dataset,

    "ground_truth": gt,
}
```

**4. Update the results dict:**

  <name>Task 2: Add SSE streaming audit endpoint</name>
  <files>mizan/backend/app/routers/audit.py</files>
  <action>
Add a new `POST /api/audit/run/stream` endpoint to `audit.py` that streams SSE progress events during the inference loop. Keep the existing `POST /api/audit/run` endpoint unchanged.

**Endpoint signature:**
```python
@router.post("/run/stream")
async def run_audit_stream(
    current_user: Annotated[User, Depends(get_current_user)],
    db: Session = Depends(get_db),
):
```

**Implementation:**

1. Import `asyncio` and `json` at top of file (add to existing imports).

2. Validate model loaded and examples exist (same guards as `run_audit()`).

3. Load all examples eagerly into memory BEFORE creating the generator:
```python
examples = db.query(ContentExample).all()
total = len(examples)
```

4. Create an `async def event_generator()` that:
   - Initializes all tracking dicts (same as Task 1's enriched loop: overall tp/fp/fn/tn, category_stats, source_stats, category_confidences, overall_confidences, false_positive_samples)
   - Records `start_time = time.perf_counter()`
   - Loops through examples. For each example:
     - Runs `model_manager.classify()` wrapped in `await asyncio.get_event_loop().run_in_executor(None, ...)` since classify is synchronous
     - Updates all tracking counters (identical logic to Task 1)
     - Every example, yields an SSE event: `f"data: {json.dumps({'progress': round((i+1)/total*100), 'current': i+1, 'total': total})}\n\n"`
   - After the loop, builds the enriched `results` dict (same construction as Task 1)
   - Caches to BiasAuditRun (same as existing `run_audit()`)
   - Commits the DB session
   - Yields a final SSE event: `f"data: {json.dumps({'done': True, 'id': str(run.id), 'computed_at': run.computed_at.isoformat(), 'total_examples': run.total_examples, 'duration_ms': run.duration_ms, 'results': results})}\n\n"`

5. Wrap the entire inference loop body inside the generator in a try/except. On exception, yield an error event: `f"data: {json.dumps({'error': str(e)})}\n\n"` and return.

6. Return `StreamingResponse(event_generator(), media_type="text/event-stream")`.

**Extract shared helper to avoid code duplication:**
Create a private function `_run_inference_loop(examples, model_manager, settings)` that yields `(i, ex, result, gt_binary, gt, predicted)` tuples. Both `run_audit()` and the SSE generator can consume this. OR — simpler: extract `_build_enriched_results(examples, model_manager, settings)` that returns the final results dict + metadata. `run_audit()` calls it directly. The SSE generator duplicates the loop (acceptable since the SSE version needs per-iteration yields).

Choose the simpler approach: the SSE generator has its own loop (duplicated logic is ~30 lines; extracting a shared yielding iterator adds complexity for marginal DRY benefit). But DO extract the post-loop results construction into a shared `_build_results_from_stats()` helper that both endpoints call.

**IMPORTANT:**
- The DB session from `get_db()` must NOT be used after the generator starts yielding. Load `examples` eagerly and pass `db` only for the final commit. Use `db.add(run); db.commit()` inside the generator before the final yield.
- Set SSE headers: `Cache-Control: no-cache`, `Connection: keep-alive` via StreamingResponse headers parameter.
