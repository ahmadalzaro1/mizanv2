# Phase 09.1: Bias Auditor Rework - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Rework the existing Bias Auditor from a basic per-category metrics viewer into a richer fairness analysis tool. Add confidence distribution histograms, per-source dataset breakdown, false positive sample viewer, SSE progress streaming, and auto-generated Arabic insight summary. No new Python dependencies. No schema migration. All new data fits in existing `BiasAuditRun.results` JSONB field.

</domain>

<decisions>
## Implementation Decisions

### Progress Experience
- SSE streaming with live progress bar using `fetch()` + `ReadableStream` (not `EventSource` — JWT Bearer auth requires custom headers)
- Real-time updates as each example is classified during the 560-example inference loop
- Progress data streamed as SSE `data:` events with JSON payload (`progress`, `current`, `total`)

### New Analysis Views
- All three new views included: confidence histograms, per-source breakdown, false positive samples
- **Confidence histograms**: small multiples 3x3 grid showing all 9 categories simultaneously. Categories with <20 samples show Arabic text note instead of histogram ("عدد العينات غير كافٍ للتوزيع")
- **Per-source breakdown**: table format with color-coded F1 cells (green/amber/red thresholds). Columns: source name, total examples, F1, precision, recall, FP rate
- **False positive samples**: up to 10 non-hate examples incorrectly classified as hate. Each shows: Arabic text (truncated ~120 chars), source dataset, confidence score with visual bar indicator. High-confidence false positives visually flagged as most concerning

### Visual Layout
- 4-tab layout: نظرة عامة (overview + existing per-category chart), توزيع الثقة (confidence histograms), مصادر البيانات (source table), الإيجابيات الكاذبة (false positive samples)
- Run/re-run button + 4 overall metric cards pinned above tabs — always visible regardless of active tab
- Histogram grid: category title (Arabic) + bars only per mini-chart, shared X-axis label at grid bottom (0% → 100%)

### Researcher Value
- Pitch angle: "Here's where the model breaks" — lead with honest weaknesses, not positive framing
- Auto-generated Arabic insight summary (2-3 sentences) at the top of the overview tab, template-based from results. Highlights: weakest category by F1, false positive rate, hardest data source
- FP viewer cards include confidence bar with visual indicator — high-confidence false positives are flagged as most concerning (model was very sure and still wrong)

### Claude's Discretion
- SSE error handling approach (show error + retry vs auto-retry)
- Whether to keep old POST /api/audit/run alongside new streaming endpoint, or replace
- Progress bar detail level (percentage + count vs percentage only)
- Tab bar visual style (underline vs pill tabs)
- CSV download scope (enhanced comprehensive vs separate downloads)
- Whether to add per-source FP rate alert alongside existing F1 < 50% weakness alert

</decisions>

<specifics>
## Specific Ideas

- The pitch should show the jury that Mizan is honest about where MARBERT fails — weakest categories and false positives are the story
- The auto-generated insight summary gives Lina (NLP researcher) a narrative she can reference in her own papers
- Confidence histograms reveal whether the model is uncertain or overconfident on specific hate types — a calibration signal
- Per-source breakdown surfaces domain adaptation failures across the 4 source datasets (JHSC, OSACT5/MLMA, L-HSAB, Let-Mi) which have different dialects and annotation schemes

<code_context>
## Existing Code Insights

### Reusable Assets
- `BiasChart.tsx`: D3 horizontal grouped bar chart with Arabic Y-axis labels, Tajawal font, RTL-safe `text-anchor: end`. Reuse pattern for new charts
- `toArabicDigits()` in `format.ts`: shared utility for Arabic number rendering in all new components
- `audit-api.ts`: existing API client with `api` shared instance (handles JWT token + base URL). Extend with SSE streaming function
- `_compute_metrics()` in `audit.py`: precision/recall/F1 from confusion counts — reuse for per-source metrics
- `CATEGORY_LABELS_AR` dict in `audit.py`: Arabic labels for all 9 hate categories — reuse in new views

### Established Patterns
- D3 charts: `useRef` + `useEffect`, clear-and-redraw on data change, SVG viewBox for responsiveness
- BiasAuditRun.results JSONB: schemaless — new keys (`per_source`, `confidence_dist`, `false_positives`) added without migration
- Backend raw SQL pattern: established in `observatory.py` for complex queries — same approach for per-source aggregation
- Tailwind color system: `mizan-navy`, `mizan-surface`, `font-tajawal` — use for all new UI elements

### Integration Points
- `POST /api/audit/run` loop in `audit.py`: single pass through 560 examples — extend to capture confidence scores, per-source stats, and FP samples in the same loop
- `BiasAuditorPage.tsx`: main page component — restructure into tabbed layout with pinned header
- `audit-api.ts` types: `AuditResults` interface needs new optional fields for backward compatibility with cached results
- `ContentExample.source_dataset` column: already exists, values are "JHSC", "OSACT5", "L-HSAB", "Let-Mi"
