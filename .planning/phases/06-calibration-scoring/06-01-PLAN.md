## Goal

### Task 1: Update _serialize_session to always compute correct_count

Change `_serialize_session()` (starts at line 56) to compute `correct_count` from items instead of reading the DB column (which is only set on session completion).

**New code:**
```python
def _serialize_session(session: TrainingSession, include_items: bool = False) -> dict:
    labeled_count = sum(1 for i in session.items if i.moderator_label is not None)
    correct_count = sum(1 for i in session.items if i.is_correct is True)
    result = {
        ...
        "labeled_count": labeled_count,
        "correct_count": correct_count,
        ...
    }
```

**Key change:** Replace `session.correct_count` (DB column, null until complete) with a live computation from items. This makes session lists show in-progress scores too.

**Note:** The `SessionHistoryList` component in TrainingPage already has access to `correct_count` from the session list API, but does not currently display it for in-progress sessions. This task only fixes the backend serialization — the session list UI is not in scope for TRAIN-06 (which is about the labeling session view, not the session list).

### Task 2: Create CalibrationScore component

**File:** `mizan/frontend/src/components/CalibrationScore.tsx`

New component that displays the live calibration score with Arabic text and color coding.

```typescript
interface CalibrationScoreProps {
  correctCount: number
  labeledCount: number
}
```

**Rendering logic:**
- If `labeledCount === 0`: Show gray placeholder "ستظهر نسبة المعايرة بعد أول تصنيف"
- If `labeledCount > 0`: Compute `percentage = Math.round((correctCount / labeledCount) * 100)` and display:
  - Arabic digits percentage: "نسبة المعايرة: ٧٠٪"
  - Color-coded badge (same thresholds as SessionSummary):
    - ≥80% → `bg-green-100 text-green-800`
    - ≥60% → `bg-amber-100 text-amber-800`
    - <60% → `bg-red-100 text-red-800`
  - Correct/labeled count in smaller text: "٧ من ١٠ صحيح"

**Important design note:** CalibrationScore divides by `labeledCount` (items labeled so far), NOT `total_items`. This is intentional — TRAIN-06 says "% agreement with ground truth" for items the moderator has actually labeled. This differs from `SessionSummary.tsx` which divides by `total_items` (showing final session score). The two formulas converge when the session is complete (all items labeled).

**toArabicDigits**: Import from the shared `format.ts` utility (Task 3).

**Visual design:**
```tsx
import { toArabicDigits } from '../lib/format'

export default function CalibrationScore({ correctCount, labeledCount }: CalibrationScoreProps) {
  if (labeledCount === 0) {
    return (
      <div dir="rtl" className="rounded-lg bg-gray-50 px-4 py-2 text-center font-tajawal text-sm text-gray-400">
        ستظهر نسبة المعايرة بعد أول تصنيف
      </div>
    )
  }

  const percentage = Math.round((correctCount / labeledCount) * 100)

  let colorClass: string
  if (percentage >= 80) {
    colorClass = 'bg-green-100 text-green-800'
  } else if (percentage >= 60) {
    colorClass = 'bg-amber-100 text-amber-800'
  } else {
    colorClass = 'bg-red-100 text-red-800'
  }

  return (
    <div dir="rtl" className={`flex items-center justify-between rounded-lg px-4 py-2 font-tajawal transition-all duration-300 ${colorClass}`}>
      <span className="text-sm font-bold">
        نسبة المعايرة: {toArabicDigits(percentage)}٪
      </span>
      <span className="text-xs opacity-75">
        {toArabicDigits(correctCount)} من {toArabicDigits(labeledCount)} صحيح
      </span>
    </div>
  )
}
```

### Task 3: Extract toArabicDigits utility

**File:** `mizan/frontend/src/lib/format.ts`

Extract the `toArabicDigits` function (currently duplicated in ProgressBar.tsx and SessionSummary.tsx) into a shared utility:

```typescript
const ARABIC_DIGITS = ['٠', '١', '٢', '٣', '٤', '٥', '٦', '٧', '٨', '٩']

export function toArabicDigits(n: number): string {
  return String(n)
    .split('')
    .map((d) => ARABIC_DIGITS[Number(d)] ?? d)
    .join('')
}
```
