### Task 1: Create Pydantic schemas for classify
**File**: `mizan/backend/app/schemas/classify.py`

```python
from pydantic import BaseModel, Field


class ClassifyRequest(BaseModel):
    text: str = Field(..., min_length=1, max_length=5000, description="Arabic text to classify")


class ClassifyProbabilities(BaseModel):
    hate: float
    not_hate: float


class ClassifyResponse(BaseModel):
    label: str  # "hate" or "not_hate"
    confidence: float  # 0.0 - 1.0
    probabilities: ClassifyProbabilities
    model_used: str  # "marbert" or "xlm-roberta"
    processing_time_ms: int


class HealthResponse(BaseModel):
    loaded: bool
    device: str | None
    marbert_loaded: bool
    xlm_loaded: bool
```

### Task 2: Create the classify router
**File**: `mizan/backend/app/routers/classify.py`

```python
from typing import Annotated
from fastapi import APIRouter, Depends, HTTPException, status
from app.core.deps import get_current_user
from app.models.user import User
from app.schemas.classify import ClassifyRequest, ClassifyResponse, HealthResponse

router = APIRouter(prefix="/api/classify", tags=["classify"])


def get_model_manager():
    """Dependency to get the global ModelManager instance."""
    from app.main import model_manager
    if model_manager is None or not model_manager.is_loaded:
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail="النماذج غير جاهزة بعد. يرجى الانتظار.",  # Models not ready yet
        )
    return model_manager


@router.post("", response_model=ClassifyResponse)
def classify_text(
    request: ClassifyRequest,
    current_user: Annotated[User, Depends(get_current_user)],
    mm=Depends(get_model_manager),
):
    """Classify Arabic text as hate/not_hate using MARBERT (or XLM-RoBERTa for code-mixed)."""
    from app.core.config import settings
    result = mm.classify(request.text, code_mixed_threshold=settings.code_mixed_threshold)
    return result


@router.get("/health", response_model=HealthResponse)
def classify_health(mm=Depends(get_model_manager)):
    """Check model loading status."""
    return mm.get_status()
```

**Design notes:**
- Uses `def` (not `async def`) — FastAPI runs it in a threadpool, keeping the event loop free while PyTorch runs inference.
- `get_model_manager` returns 503 if models aren't loaded yet (handles requests during startup).
- Auth required on classify, health endpoint also behind model check.

### Task 3: Update main.py with lifespan and router registration
**File**: `mizan/backend/app/main.py`

Replace the current `main.py` with lifespan-based model loading:

```python
import os
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from app.routers import auth, export, training, classify
from app.services.ml_models import ModelManager

# Global model manager — accessed by classify router dependency
model_manager: ModelManager | None = None


@asynccontextmanager
async def lifespan(app: FastAPI):
    """Load ML models at startup, release at shutdown."""
    global model_manager

    # Set MPS fallback for unsupported ops
    os.environ.setdefault("PYTORCH_ENABLE_MPS_FALLBACK", "1")

    from app.core.config import settings

    mm = ModelManager()
    mm.load_models(
        marbert_id=settings.marbert_model_id,
        xlm_id=settings.xlm_model_id,
    )
    model_manager = mm

    yield  # Server is running, handling requests

    # Cleanup
    model_manager = None


app = FastAPI(title="Mizan API", version="0.2.0", lifespan=lifespan)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(auth.router)
app.include_router(export.router)
app.include_router(training.router)
app.include_router(classify.router)


@app.get("/health")
def health_check():
    return {"status": "ok", "service": "mizan-backend"}
```

**Key changes from current main.py:**
- Added `lifespan` context manager (replaces nothing — no startup events existed before)
- Added `model_manager` global variable
- Added `classify.router` registration
- Version bump to 0.2.0
- Set `PYTORCH_ENABLE_MPS_FALLBACK=1` for MPS compatibility

### Task 5: End-to-end verification

**Step 1 — Start the backend:**
```bash
cd mizan/backend
uvicorn app.main:app --reload
```

Wait for startup log indicating models are loaded (~1-5 min first time for download).

**Step 2 — Test health endpoint:**
```bash
curl http://localhost:8000/api/classify/health \
  -H "Authorization: Bearer <jwt_token>"
```

Expected:
```json
{
  "loaded": true,
  "device": "mps",
  "marbert_loaded": true,
  "xlm_loaded": true
}
```

**Step 3 — Test Arabic classification (MARBERT path):**
```bash
curl -X POST http://localhost:8000/api/classify \
  -H "Authorization: Bearer <jwt_token>" \
  -H "Content-Type: application/json" \
  -d '{"text": "كل العرب ارهابيين ولازم يتطردوا"}'
```

Expected: `label: "hate"`, `model_used: "marbert"`, response < 3000ms.

**Step 4 — Test code-mixed classification (XLM-RoBERTa path):**
```bash
curl -X POST http://localhost:8000/api/classify \
  -H "Authorization: Bearer <jwt_token>" \
  -H "Content-Type: application/json" \
  -d '{"text": "These people are disgusting يلعن ابوهم they should be deported"}'
```

Expected: `model_used: "xlm-roberta"` (Latin chars > 30%), valid classification.

**Step 5 — Test error cases:**
- Empty text: `{"text": ""}` → 422
- No auth header → 401
- Very long text (5000+ chars) → 422

**Step 6 — Check timing:**
All single-text classify requests should return in < 3 seconds (expect ~50-200ms).
