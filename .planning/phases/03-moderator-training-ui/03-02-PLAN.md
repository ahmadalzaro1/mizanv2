### Task 1: Create SQLAlchemy models
**File**: `mizan/backend/app/models/training.py`

Two new models. **Important**: Use `UUID(as_uuid=True)` for all UUID columns to match existing codebase pattern (see `ContentExample`, `User`, `Institution`).

```python
class SessionStatus(str, enum.Enum):
    in_progress = "in_progress"
    completed = "completed"

class ModeratorLabel(str, enum.Enum):
    hate = "hate"
    not_hate = "not_hate"

class TrainingSession(Base):
    __tablename__ = "training_sessions"
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    user_id = Column(UUID(as_uuid=True), ForeignKey("users.id"), nullable=False)
    institution_id = Column(UUID(as_uuid=True), ForeignKey("institutions.id"), nullable=True)
    status = Column(Enum(SessionStatus), default=SessionStatus.in_progress)
    total_items = Column(Integer, default=20)
    correct_count = Column(Integer, nullable=True)  # set on completion
    created_at = Column(DateTime, default=datetime.utcnow)
    completed_at = Column(DateTime, nullable=True)
    # relationships
    user = relationship("User")
    items = relationship("SessionItem", back_populates="session", order_by="SessionItem.position")

class SessionItem(Base):
    __tablename__ = "session_items"
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    session_id = Column(UUID(as_uuid=True), ForeignKey("training_sessions.id", ondelete="CASCADE"))
    content_example_id = Column(UUID(as_uuid=True), ForeignKey("content_examples.id"))
    position = Column(Integer, nullable=False)  # 1-based
    moderator_label = Column(Enum(ModeratorLabel), nullable=True)
    moderator_hate_type = Column(Enum(HateType), nullable=True)  # reuse from content_example.py
    is_correct = Column(Boolean, nullable=True)
    labeled_at = Column(DateTime, nullable=True)
    # relationships
    session = relationship("TrainingSession", back_populates="items")
    content_example = relationship("ContentExample")
```

Update `mizan/backend/app/models/__init__.py` to import the new models.

**Note on `institution_id`:** This column is nullable because super_admins have `institution_id=None`. In practice, only moderators/admins will create training sessions. The nullable FK is consistent with the User model pattern.

### Task 3: Create Pydantic schemas and training router
**Files**: `mizan/backend/app/schemas/training.py`, `mizan/backend/app/routers/training.py`, `mizan/backend/app/main.py`

**Schemas (`schemas/training.py`):**
```python
# Request schemas
class SubmitLabelRequest(BaseModel):
    moderator_label: Literal["hate", "not_hate"]
    moderator_hate_type: str | None = None  # required if moderator_label == "hate"

# Response schemas
class ContentExampleBrief(BaseModel):
    id: UUID
    text: str
    source_dataset: str
    dialect: str | None

class SessionItemResponse(BaseModel):
    id: UUID
    position: int
    content_example: ContentExampleBrief
    moderator_label: str | None
    moderator_hate_type: str | None
    is_correct: bool | None
    ground_truth_label: str | None  # only revealed AFTER moderator labels (anti-cheat)
    ground_truth_hate_type: str | None  # only revealed AFTER moderator labels
    labeled_at: datetime | None

class TrainingSessionResponse(BaseModel):
    id: UUID
    status: str
    total_items: int
    labeled_count: int  # computed: count of items where moderator_label is not null
    correct_count: int | None
    created_at: datetime
    completed_at: datetime | None

class TrainingSessionDetailResponse(TrainingSessionResponse):
    items: list[SessionItemResponse]

class SessionListResponse(BaseModel):
    sessions: list[TrainingSessionResponse]
```

**Key design:** `ground_truth_label` and `ground_truth_hate_type` in `SessionItemResponse` are only populated AFTER the moderator has submitted their label for that item. Before labeling, these fields are `null` (prevents cheating by looking ahead). This is enforced at the API level in the response serialization.

**Router (`routers/training.py`):**
```python
router = APIRouter(prefix="/api/training", tags=["training"])
```

**Endpoint 1: POST /api/training/sessions**
- Auth: `get_current_user` (any authenticated user)
- Logic:
  1. Query `content_examples` for 20 random rows NOT already assigned to this user in any session
  2. If fewer than 20 unassigned examples remain, use however many are left (min 1, else 400 error with message "لا توجد أمثلة متاحة")
  3. Create `TrainingSession` with `user_id=current_user.id`, `institution_id=current_user.institution_id`
  4. Create `SessionItem` rows with `position` 1–N
  5. Return `TrainingSessionDetailResponse`

Random selection query:
```python
from sqlalchemy import func
used_ids = db.query(SessionItem.content_example_id).join(TrainingSession).filter(
    TrainingSession.user_id == current_user.id
).subquery()
examples = db.query(ContentExample).filter(
    ~ContentExample.id.in_(used_ids)
).order_by(func.random()).limit(20).all()
```

**Endpoint 2: GET /api/training/sessions/{session_id}**
- Auth: `get_current_user` — verify `session.user_id == current_user.id` (else 404)
- Returns: `TrainingSessionDetailResponse` with all items
- Ground truth is revealed only for items where `moderator_label IS NOT NULL`

**Endpoint 3: PUT /api/training/sessions/{session_id}/items/{item_id}**
- Auth: `get_current_user` — verify session belongs to this user
- Body: `SubmitLabelRequest`
- Validates: if `moderator_label == "hate"`, `moderator_hate_type` is required (else 422)
- Computes `is_correct` using **binary mapping**:
  - Ground truth `hate` OR `offensive` → treated as "hate" for scoring
  - Ground truth `not_hate` OR `spam` → treated as "not_hate" for scoring
  - Moderator's `hate` matches ground truth hate/offensive → correct
  - Moderator's `not_hate` matches ground truth not_hate/spam → correct
  - Otherwise → incorrect
  - **Rationale:** `offensive` content is harmful speech, so matching it to `hate` rewards moderators who catch it. `spam` is not harmful, so `not_hate` is correct. This mapping may be revisited in Phase 6 (calibration scoring).
- Sets `labeled_at = now()`
- After saving, check if ALL items in session are labeled. If yes:
  - Set `session.status = "completed"`
  - Set `session.completed_at = now()`
  - Set `session.correct_count = count of is_correct=True items`
- Returns updated `SessionItemResponse` (now includes ground truth)

**Endpoint 4: GET /api/training/sessions**
- Auth: `get_current_user`
- Returns: list of sessions for this user, ordered by `created_at` desc
- Includes computed `labeled_count` for each session

**Register router in `main.py`:**
```python
from app.routers import training
app.include_router(training.router)
```

### Task 4: Test endpoints manually
Using curl or httpie against the running backend:
1. Login as demo moderator → get JWT token
2. POST /api/training/sessions → get session with 20 items
3. GET /api/training/sessions/{id} → see items with null labels, null ground_truth
4. PUT /api/training/sessions/{id}/items/{item_id} → submit a label, verify ground truth revealed
5. Verify `is_correct` logic: submit `hate` on an `offensive` ground truth → should be correct
6. Submit all 20 → verify session status = "completed" and correct_count set
7. POST /api/training/sessions again → verify new session excludes previous examples

## Success Criteria
1. `alembic upgrade head` creates both tables without errors
2. POST /api/training/sessions returns a session with 20 items
3. Each item has Arabic text, position, and null labels before labeling
4. PUT label submission computes is_correct correctly and reveals ground truth
5. `offensive` ground truth + moderator `hate` → is_correct=True
6. After all 20 labeled, session auto-completes with correct_count
7. Second session creation excludes examples from first session
