## Tasks

### Task 1: Create TrainingSession page (labeling view)
**File**: `mizan/frontend/src/pages/TrainingSession.tsx`

The main labeling interface. Route: `/train/sessions/:sessionId`

**State management** — use `useReducer` for session state:
```typescript
interface SessionState {
  session: TrainingSession | null
  currentIndex: number  // 0-based index into items array
  isSubmitting: boolean
  error: string | null
}
```

**View logic for each item:**
- If item has NO `moderator_label` → show TweetCard + LabelSelector (labeling mode)
- If item HAS `moderator_label` → show TweetCard + FeedbackReveal (review mode)
  - In review mode, LabelSelector can be shown again (pre-filled) to allow changing the answer

**Navigation:**
- "السابق" (Previous) button: go to `currentIndex - 1` (disabled at index 0)
- "التالي" (Next) button: go to `currentIndex + 1` (disabled if current item unlabeled)
- ProgressBar shows position and labeled count
- Navigation uses buttons only (no keyboard shortcuts — simpler for Arabic interface)

**Submit flow:**
1. Moderator selects label → clicks "إرسال" (Submit)
2. `isSubmitting = true`, call `submitLabel()` API
3. Update local state with response (now has `ground_truth`, `is_correct`)
4. `isSubmitting = false`, show FeedbackReveal component
5. Moderator clicks "التالي" (Next) → advance to next item

**Re-labeling (back navigation):**
- When navigating back to a labeled item, show the feedback view
- Include a "تغيير الإجابة" (Change answer) button that switches back to labeling mode
- Re-submitting updates the label via the same PUT endpoint

**Session completion:**
- When all items are labeled and user clicks Next on the last item:
  - Navigate to `/train/sessions/${sessionId}/summary`
- If navigating directly to a completed session's labeling view, redirect to summary

**Load on mount:**
- Get `sessionId` from route params
- Call `getSession(sessionId)` on mount
- If session not found or doesn't belong to user → redirect to `/train`
- Find the first unlabeled item and set it as `currentIndex`

### Task 2: Create TrainingPage (landing) and SessionSummary page
**Files**: `mizan/frontend/src/pages/TrainingPage.tsx`, `mizan/frontend/src/pages/SessionSummary.tsx`

**TrainingPage** — the `/train` route landing page:

If moderator has no sessions:
- Welcome message: "مرحباً بك في التدريب"
- Instructions: "ستعرض عليك ٢٠ تغريدة عربية. صنّف كل واحدة: خطاب كراهية أو ليس كراهية."
- "ابدأ التدريب" (Start Training) button — large, prominent, navy background

If moderator has existing sessions:
- "ابدأ جلسة جديدة" (Start New Session) button at top
- SessionHistoryList below

On "Start Training" click:
1. Show loading state on button
2. Call `createSession()` API
3. On success: `navigate(`/train/sessions/${session.id}`)`
4. On error: show Arabic error (e.g., "لا توجد أمثلة متاحة")

**SessionSummary** — route: `/train/sessions/:sessionId/summary`

Summary card:
- Large accuracy display using Arabic text: e.g., "١٤ من ٢٠" with percentage
- Visual score indicator:
  - ≥80%: green background with "ممتاز!" (Excellent!)
  - 60-79%: amber background with "جيد" (Good)
  - <60%: red background with "يحتاج تحسين" (Needs improvement)

Item breakdown:
- Scrollable list of all 20 items showing:
  - Tweet text (truncated to ~50 chars with `...`)
  - Moderator's label vs ground truth
  - Correct/incorrect icon (green check / red X)

Actions:
- "ابدأ جلسة جديدة" (Start new session) button
- "العودة إلى لوحة التحكم" (Back to dashboard) link

On mount: call `getSession(sessionId)`. If session not completed, redirect to active labeling view.

### Task 3: Create SessionHistoryList component
**File**: `mizan/frontend/src/components/SessionHistoryList.tsx`

Displays past sessions in a card list:

Each session card shows:
| Field | Display |
|-------|---------|
| Date | Arabic formatted (e.g., "٢ مارس ٢٠٢٦") |
| Status | "مكتملة" (completed) / "قيد التقدم" (in progress) |
| Score | "١٤/٢٠ (٧٠٪)" or "—" if not complete |
| Action | "استمرار" (Continue) for in_progress, "مراجعة" (Review) for completed |

```tsx
interface SessionHistoryListProps {
  sessions: TrainingSession[]
}
```

Links:
- In-progress → `/train/sessions/{id}` (continues labeling)
- Completed → `/train/sessions/{id}/summary`
