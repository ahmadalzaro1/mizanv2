## Goal

Playwright is installed, configured, and can launch Chromium headless against the running dev servers. A `globalSetup` script seeds the database and writes a `storageState` auth file so all subsequent test specs start authenticated. Running `npm run test:e2e` from `mizan/frontend/` launches the test suite.

## Prerequisites

- Phase 8 complete (all frontend routes and backend endpoints working)
- Docker Compose stack running (`docker compose up`) with backend at `localhost:8000` and frontend at `localhost:5173`
- Seed scripts (`seed.py`, `seed_content.py`, `seed_jhsc.py`) are idempotent and ready

## must_haves

- `@playwright/test` installed as a devDependency in `mizan/frontend/package.json`
- Chromium browser installed via `npx playwright install chromium`
- `playwright.config.ts` exists with correct `testDir`, `baseURL`, `storageState`, `globalSetup`, single worker, Chromium-only project, 30s default timeout
- `tsconfig.e2e.json` allows Playwright test files to compile without conflicting with the Vite/React tsconfig
- `global-setup.ts` invokes seed scripts via `child_process.execSync`, calls `POST /auth/login`, and writes `auth-state.json` with the `mizan_token` localStorage entry
- `auth-state.json` is git-ignored
- `playwright-report/` and `test-results/` directories are git-ignored
- `npm run test:e2e` script exists in `package.json`
- A smoke test (`npx playwright test --list`) reports discovered test files without errors

### Task 2: Create Playwright config

**File:** `mizan/frontend/playwright.config.ts`

```typescript
import { defineConfig } from '@playwright/test'

const BASE_URL = process.env.BASE_URL ?? 'http://localhost:5173'
const API_URL = process.env.API_URL ?? 'http://localhost:8000'

export default defineConfig({
  testDir: './e2e',
  timeout: 30_000,
  expect: {
    timeout: 10_000,
  },
  fullyParallel: false,
  workers: 1,
  retries: 0,
  reporter: [['html', { open: 'never' }], ['list']],
  globalSetup: './e2e/global-setup.ts',

  use: {
    baseURL: BASE_URL,
    storageState: './e2e/fixtures/auth-state.json',
    trace: 'retain-on-failure',
    screenshot: 'only-on-failure',
    locale: 'ar',
    viewport: { width: 1280, height: 720 },
  },

  projects: [
    {
      name: 'chromium',
      use: {
        browserName: 'chromium',
      },
    },
  ],
})
```

**Key settings:**
- `workers: 1` — sequential execution prevents DB race conditions
- `storageState` — all tests start with `mizan_token` pre-set in localStorage (the login UI test overrides this per-file)
- `trace: 'retain-on-failure'` — only captures trace on failed tests
- `locale: 'ar'` — Arabic locale for the browser context
- `globalSetup` — seeds DB and creates auth state before any test runs
- `retries: 0` — failures must be investigated, not retried blindly
- No `webServer` block — assumes servers are already running (Docker Compose or manual `npm run dev`)

### Task 3: Create TypeScript config for E2E tests

**File:** `mizan/frontend/tsconfig.e2e.json`

The existing `tsconfig.json` uses `moduleResolution: "bundler"` and `allowImportingTsExtensions: true` which are incompatible with Playwright's Node-based test runner. Create a separate config:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "strict": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "types": ["node"]
  },
  "include": ["e2e/**/*.ts", "playwright.config.ts"]
}
```

**Notes:**
- `moduleResolution: "node"` — Playwright uses Node module resolution, not Vite's bundler resolution
- Does NOT include `src/` — test files should not import from the app source (they interact via the browser)
- `types: ["node"]` — for `child_process`, `fs`, `path` usage in `global-setup.ts`
- No `allowImportingTsExtensions` — not needed for Playwright files

### Task 4: Create globalSetup

**File:** `mizan/frontend/e2e/global-setup.ts`

This file runs once before the entire test suite. It:
1. Invokes the three seed scripts to ensure test data exists
2. Calls `POST /auth/login` to get a JWT
3. Writes a `storageState` JSON file with the token in localStorage format

```typescript
import { execSync } from 'node:child_process'
import { writeFileSync, mkdirSync } from 'node:fs'
import { resolve } from 'node:path'

const API_URL = process.env.API_URL ?? 'http://localhost:8000'
const FRONTEND_URL = process.env.BASE_URL ?? 'http://localhost:5173'
const AUTH_STATE_PATH = resolve(__dirname, 'fixtures', 'auth-state.json')

// Demo admin credentials (from seed.py — idempotent)
const TEST_EMAIL = 'demo-admin@mizan.local'
const TEST_PASSWORD = 'demo_admin_2026'

async function globalSetup() {
  console.log('[global-setup] Seeding database...')

  // Run seed scripts. Use Docker Compose exec if available, else direct Python.
  // Seeds are idempotent — safe to run on every test invocation.
  const seedCommands = [
    'docker compose exec -T backend python scripts/seed.py',
    'docker compose exec -T backend python scripts/seed_content.py',
    'docker compose exec -T backend python scripts/seed_jhsc.py',
  ]

  for (const cmd of seedCommands) {
    try {
      execSync(cmd, {
        cwd: resolve(__dirname, '..', '..'),  // mizan/ directory
        stdio: 'pipe',
        timeout: 120_000,  // JHSC seed can take a while on first run
      })
      console.log(`[global-setup] OK: ${cmd.split(' ').pop()}`)
    } catch (err) {
      console.warn(`[global-setup] Seed command failed (may already be seeded): ${cmd}`)
      // Non-fatal — data may already exist from a prior run
    }
  }

  console.log('[global-setup] Obtaining auth token...')

  // POST /auth/login to get JWT
  const loginResponse = await fetch(`${API_URL}/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email: TEST_EMAIL, password: TEST_PASSWORD }),
  })

  if (!loginResponse.ok) {
    throw new Error(
      `[global-setup] Login failed: ${loginResponse.status} ${await loginResponse.text()}`
    )
  }

  const { access_token } = (await loginResponse.json()) as { access_token: string }

  if (!access_token) {
    throw new Error('[global-setup] No access_token in login response')
  }

  console.log('[global-setup] Token obtained. Writing auth state...')

  // Write storageState JSON — Playwright reads this to inject localStorage
  const storageState = {
    cookies: [],
    origins: [
      {
        origin: FRONTEND_URL,
        localStorage: [
          { name: 'mizan_token', value: access_token },
        ],
      },
    ],
  }

  // Ensure fixtures directory exists
  mkdirSync(resolve(__dirname, 'fixtures'), { recursive: true })

  writeFileSync(AUTH_STATE_PATH, JSON.stringify(storageState, null, 2), 'utf-8')

  console.log(`[global-setup] Auth state written to ${AUTH_STATE_PATH}`)
}

export default globalSetup
```

**Key details:**
- `cwd` for `execSync` points to `mizan/` (the Docker Compose root)
- `timeout: 120_000` — the JHSC seed (403K rows) can take time on first run
- `stdio: 'pipe'` — suppresses seed script stdout from polluting test output
- Seed failures are non-fatal warnings (data may already exist)
- Login failure IS fatal (tests cannot run without auth)
- `mizan_token` key matches the exact localStorage key used by `api.ts`
- Fixtures directory created automatically on first run
