**Highlighting logic:**
- When `highlights` is provided, split the text and wrap matched words in `<mark>` elements
- For `type === 'hate'`: use `bg-amber-200 text-amber-900 rounded px-0.5` (orange/amber highlight)
- For `type === 'not_hate'`: use `bg-green-200 text-green-900 rounded px-0.5` (green highlight)
- When `highlights` is not provided (unlabeled items), render text as-is (unchanged behavior)

**Word matching approach:**
- Simple string matching — for each trigger word, find its occurrence in the text
- Use a function that builds React elements by scanning through the text
- Handle Arabic text properly (no case sensitivity needed, Arabic has no case)
- If a trigger word appears multiple times, highlight only the first occurrence

```typescript
/** Strip Arabic diacritics (tashkeel) for matching — display preserves original. */
function stripDiacritics(s: string): string {
  return s.normalize('NFD').replace(/[\u064B-\u065F\u0670]/g, '')
}

function highlightText(
  text: string,
  words: string[],
  className: string
): React.ReactNode {
  if (words.length === 0) return text

  // Normalize both text and trigger words to handle diacritics mismatch
  // (tokenizer strips diacritics but tweet text may have tashkeel)
  const normalizedText = stripDiacritics(text)
  const normalizedWords = words.map(stripDiacritics)

  // Build regex from normalized trigger words, escaped for special chars
  const escaped = normalizedWords.map(w => w.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'))
  const pattern = new RegExp(`(${escaped.join('|')})`, 'g')

  // Find match positions in normalized text, apply to original text
  const parts: React.ReactNode[] = []
  let lastIndex = 0
  let match: RegExpExecArray | null

  while ((match = pattern.exec(normalizedText)) !== null) {
    if (match.index > lastIndex) {
      parts.push(text.slice(lastIndex, match.index))
    }
    const matchEnd = match.index + match[0].length
    parts.push(
      <mark key={match.index} className={className}>
        {text.slice(match.index, matchEnd)}
      </mark>
    )
    lastIndex = matchEnd
  }
  if (lastIndex < text.length) {
    parts.push(text.slice(lastIndex))
  }

  return parts.length > 0 ? parts : text
}
```

**Key details:**
- Strips Arabic diacritics (fatha, kasra, damma, shadda, etc.) from both text and trigger words before matching
- Displays the original text with diacritics preserved — only matching logic normalizes
- Uses character positions from the normalized text to slice the original text

**Important:** The TweetCard already uses `dir="rtl"` and `font-tajawal` — highlights inherit these. No RTL changes needed.
